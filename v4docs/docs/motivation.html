Motivation

What It Is And Not

V4 only focuses on the transformation of XML using XSLT.

How incoming data is provided and what happens to the transformed data going out are not the concerns of V4.

Motivation

The goal of V4 is to create a unifying model where the domain is well known amongst the solutions analysts, business analysts, and developers.

This is achieved by using a language structured around the domain model.

Domain Model Usage

Business analyst

From the IRB study type copy the attribute value with the caption "Foo" to the CRMS study type with the caption "Bar".

Developer

  <Attribute Caption="Bar" Value="XYZ">
</WebService>

Solutions analyst

The CRMS study "1234" with the caption "Bar" should have a value of "ABC" but the value is "XYZ".
